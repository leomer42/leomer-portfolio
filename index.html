AI-Powered-Chatbot

  • Feb/2018 — Mar/2018

      • This is an AI chatbot that leverages Django , OpenAI and Rasa Framework , for NLP

  • Key Skills:

    • Pre-trained Model: Used OpenAi's GPT-3.5-Turbo. GPT-3.5, accessed via the OpenAI API, handles complex or conversational responses, creating human-like interactions.

    • Backend API Integration: Used Django. Django was used to set up the backend of the chatbot application. It handles API endpoint management, and serves as the central platform where interactions and workflows are coordinated. For instance, when Rasa recognizes a question that requires an open-ended answer, it can route it through Django to the OpenAI API, which then returns the generated response.

    • Framework: Used Rasa. Rasa is responsible for intent recognition and dialogue management, allowing the chatbot to interpret user inputs and handle structured flows.

    • Third-Party API Integration: Used OpenAi API.

  • How to work:

    • User Input:
      A user type prompt with the chatbot through below input area.
      The prompt message is sent to Django, which serves as the communication hub.

    • Intent Recognition with Rasa:
      Django forwards the user prompt message to Rasa, which processes it to recognize the intent and extract relevant entities (e.g., “Rasa,” ).
      Based on the intent, Rasa determines if it can handle the response with pre-defined content or actions.

    • Response Handling:
      If Rasa has a predefined answer or structured flow for the intent, it generates the response and sends it back to Django, which forwards it to the user.
      For complex or ambiguous queries, Rasa can defer the response to GPT-3.5, sending the message through Django to OpenAI’s API.
      GPT-3.5 generates a detailed response based on the input and sends it back to Django. Django then logs the response in the conversation history and forwards it to the user.

    • Contextual Management (Django):
      Django manages the user’s session and contextual flow, ensuring a coherent conversation across multiple interactions.
      By tracking the context, Django ensures that responses from Rasa and GPT-3.5 align smoothly with previous interactions.

    • Delivery to the User:
      The final response, whether from Rasa or GPT-3.5, is sent to the user via Django’s API. This response appears on the chatbot’s frontend.

Voice Chatbot

  • Sep/2019 — Oct/2019

      • Chatbot that get it in the form of speech in multiple languages all over the world

  • Key Skills:

    • Google’s TTS API: convert the Dialogflow response text into an audio file.

    • Integrate Dialogflow: sends user text to Dialogflow, receives the response, and returns it.

    • Flask Routing: Create a route that receives user messages, sends them to Dialogflow, converts the response to speech, and returns both text and audio.

  • How to work:

    • User Input: The user types a prompt message in the text input and clicks Enter key.

    • Flask Receives the Message: The frontend sends the message to the Flask server, which processes the input.

    • Dialogflow Processes Intent: The Flask backend sends the user’s message to Dialogflow, where it’s analyzed to understand the intent.
      Based on the intent, Dialogflow generates a text response.

    • Convert Response to Audio: The text response from Dialogflow is sent to Google’s Text-to-Speech API, which converts it to audio and saves it to the server.

Tour Guide

  • Jan/2023 — Sep/2023

      • AI Tour Guide offers an interactive virtual tour experience using AI-powered text-to-speech and speech-to-text functionalities for enhanced user engagement.

  • Key Features:

    • Interactive Tour Guide: Provides real-time narrations about various locations using text-to-speech capabilities.

    • User Interaction: Allows users to ask questions about the location and receive relevant answers through speech-to-text and natural language processing.

    • Easy-to-Use Interface: Features a user-friendly interface for seamless interaction with the AI guide.

    • Continuous Integration and Deployment: Ensures that updates and improvements are smoothly integrated and deployed, maintaining a high-quality user experience.

    • Backend Integration: Utilizes a robust backend system to handle user queries and integrate with various TTS and NLP models.

    • Frontend Design: Implements a clean and intuitive frontend framework to facilitate user interaction and navigation.

    • Deployment and Testing: Includes a comprehensive deployment setup with CI/CD pipelines and thorough testing to ensure reliability and performance.

  • Project Purpose:

    • The AI Tour Guide project is designed to offer an interactive and AI-powered virtual tour guide experience. By integrating advanced text-to-speech (TTS) and speech-to-text (STT) functionalities, the application provides informative and engaging narrations about various locations. It allows users to interact with the guide by asking questions and receiving detailed responses, making their exploration of different places both educational and entertaining.

  • Motivation

    • The motivation behind this project stems from the desire to enhance the experience of exploring new locations through technology. Traditional tour guides and static information can be limiting, and this project aims to leverage AI to create a dynamic and responsive tour guide. By using state-of-the-art natural language processing and speech synthesis technologies, the AI Tour Guide seeks to bridge the gap between digital and physical exploration, providing a richer, more interactive experience for users.
